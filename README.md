ჩემი თემა არის "ავტომატიზირებული სისტემები და მათი გავლენა სამუშაო პროცესების ეფექტურობაზე ციფრულ გარემოში". 
ეს რეპოზიტორია წარმოადგენს პროექტის ფარგლებში შემუშავებულ ავტომატიზირებულ ვებ-სისტემას. 
პროექტის მიზანია ბიზნეს პროცესების ავტომატიზაცია, რუტინული სამუშაოს შემცირება, მონაცემთა ეფექტური მართვა და ადამიანური შეცდომების მინიმუმამდე დაყვანა.

პროექტის მოდულებია: Dashboard - ბიზნეს პროცესების პირველადი ანალიტიკა მონიტორინგისთვის.
Conversations - სოციალური ქსელიდან შემოსული შეტყობინებების ერთიან სივრცეში მიღება და კლიენტებთან სწრაფი კომუნიკაცია.
Automation - კომპლექსური ინსტრუმენტი ავტომატიზირებული სამუშაო ნაკადების (Workflows) შესაქმნელად (მაგ. ავტომატური ელფოსტის გაგზავნა ფორმის შევსებისას).
Opportunities - გაყიდვების პროცესის ეტაპებად დაყოფა (ახალი ლიდი, მოლაპარაკება, შეთავაზება, წარმატებული და ა.შ.) და პროგრესის კონტროლი.

გამოყენებული ტექნოლოგიები:
Front-end - HTML, CSS, JavaScript.
Back-end - Python, Flask (ვებ-ფრეიმვორკი API endpoint-ების ასაგებად).
Database - MSSQL (მომხმარებელთა მონაცემებისა შესანახად).
Agile მეთოდოლოგია.

პროექტის არქიტექტურულ საფუძვლად ავარჩიე კლიენტ-სერვერული მოდელი. ეს მიდგომა საშუალებას გვაძლევს მკაფიოდ გავმიჯნოთ Front-end და Back-end.
მონაცემთა ნაკადის "მოძრაობა" იწყება კლიენტის შრეზე, სადაც მომხმარებელი HTML, CSS და JavaScript-ით აგებული 
ინტერფეისის საშუალებით ასრულებს ქმედებას (მაგ. რეგისტრაციას, შესვლას), რაც ააქტიურებს ასინქრონულ HTTP მოთხოვნას და ინფორმაციას JSON ფორმატში 
გზავნის სერვერზე. მიღებულ მონაცემებს ამუშავებს Python-ის ვებ-ფრეიმვორკი Flask. წარმატებული ვალიდაციის შემთხვევაში, ინფორმაცია საბოლოოდ ისახება და ინახება 
Microsoft SQL Server მონაცემთა ბაზაში.

API Endpoints: 
1. რეგისტრაცია - Endpoint: /register, მეთოდი: POST, ახალი მომხმარებლის რეგისტრაცია, კოდი ამოწმებს ველების შევსებას, ელ-ფოსტის ფორმატს და პაროლის სიგრძეს.
მონაცემთა ბაზაში დუბლირების თავიდან ასაცილებლად ხდება მეილის გადამოწმება, რის შემდეგაც პაროლი ჰეშირდება და ინფორმაცია ინახება ბაზაში.
2. API სტატუსის შემოწმება - Endpoint: /, API სერვერის მუშაობის შესამოწმება.
3. ავტორიზაცია - Endpoint: /login, მეთოდი: POST, ავტორიზაცია ელ-ფოსტისა და პაროლის მეშვეობით. შეყვანილი პაროლი კონვერტირდება ჰეშად და ედარება 
ბაზაში არსებულ ჩანაწერს.
4. სესიის დასრულება - Endpoint: /logout, მეთოდი: POST, მიმდინარე სესიის გასუფთავება, რის შედეგადაც მომხმარებელი გამოდის სისტემიდან.
5. მიმდინარე სესიის შემოწმება - Endpoint: /me, მეთოდი: GET, ამოწმებს, არის თუ არა მომხმარებელი მიმდინარე ეტაპზე ავტორიზებული სერვერზე.
6. მომხმარებელთა სია - Endpoint: /users, მეთოდი: GET, უკავშირდება AutomationDB მონაცემთა ბაზას და SQL მოთხოვნის საშუალებით
გამოაქვს ყველა რეგისტრირებული მომხმარებლის სია.
